## 二、最推荐刊物详细分析

### 🔥 Top 1: IEEE Transactions on Affective Computing (TAC)

```
推荐指数: ⭐⭐⭐⭐⭐ (最强推荐)

为什么最适合：
1. 【方向完美匹配】
   - 情感计算核心方向
   - 社交关系中的情感分析
   - 人机交互中的情感支持

2. 【心理学背景加分】
   - 项目已嵌入 Gottman、依恋理论
   - 心理学理论是 TAC 非常重视的

3. 【RLHF创新点】
   - 从人类偏好学习情感决策
   - 情感对齐（Emotional Alignment）
   - 符合 TAC 前沿趋势

4. 【用户实验重视】
   - TAC 重视真实用户验证
   - 100人用户实验足够

投稿要求：
- Regular Paper: 8-14页
- 需要心理学理论基础
- 需要用户实验验证
- 可随时投稿

审稿周期：6个月左右

预期接收率：中等偏高（如果用户实验充分）
```

### 🔥 Top 2: CSCW (ACM Conference on Computer-Supported Cooperative Work)

```
推荐指数: ⭐⭐⭐⭐⭐ (强烈推荐)

为什么适合：
1. 【社交计算核心】
   - CSCW 专注社交场景技术
   - 人际关系管理是核心议题

2. 【协作视角】
   - 人-AI协作决策支持
   - 符合 CSCW 理念

3. 【用户研究重视】
   - CSCW 极重视用户研究
   - 社会影响分析加分

投稿周期：
- 每年一次会议
- 通常 2月截稿，11月开会

优势：
- 社交场景完美匹配
- 用户实验权重高
- 可积累会议论文

劣势：
- 需要等固定周期
- 竞争激烈
```

### 🔥 Top 3: ACM TOCHI (Transactions on Computer-Human Interaction)

```
推荐指数: ⭐⭐⭐⭐⭐ (强烈推荐)

为什么适合：
1. 【HCI顶刊】
   - 人机交互顶级期刊
   - 影响因子约5.0

2. 【人机协同】
   - Human-AI Collaboration
   - 决策支持系统

3. 【用户研究重视】
   - TOCHI 极重视用户实验
   - 需要详细用户研究部分

投稿要求：
- 可随时投稿
- 需要 15-20页完整论文
- 用户实验部分占比大

审稿周期：4-8个月

预期接收率：中等（需要充分用户实验）
```

### 🔥 Top 4: ICMI (International Conference on Multimodal Interaction)

```
推荐指数: ⭐⭐⭐⭐⭐ (高推荐)

为什么适合：
1. 【多模态情感】
   - 项目支持文本+表情+语音多模态
   - ICMI 是多模态交互核心会议

2. 【情感计算方向】
   - ICMI 有专门的情感计算轨道
   - 情感识别+决策支持

3. 【中等难度】
   - 比 CHI/CSCW 竞争稍低
   - 更容易接收

投稿周期：
- 每年一次
- 通常 6-7月截稿

优势：
- 多模态方向匹配
- 中等接收难度
- 可积累论文

劣势：
- 需要等周期
- 影响力略低于顶会
```

## 四、投稿策略建议

### 策略A: 直接投顶刊（适合时间充足）

```
推荐路径：
IEEE TAC（首选）→ 如果被拒 → TOCHI → 如果被拒 → ICMI

时间规划：
- 3个月准备（用户实验）
- 投稿 TAC
- 等待 6个月审稿
- 如果被拒，修改后投下一个

总时间：9-15个月
成功率：中等
```

### 策略B: 先发会议积累（适合稳妥路线）

```
推荐路径：
ICMI/UMAP → 积累引用 → 再投 TAC/TOCHI

时间规划：
- 2个月准备
- 投 ICMI（中等难度）
- 会议发表积累引用
- 扩充后投顶刊

总时间：12-18个月
成功率：高
```

### 策略C: 并行投稿（适合时间紧张）

```
推荐路径：
同时准备 TAC + ICMI + UMAP

注意：
- 期刊不能同时投多个
- 会议可以同时投多个

策略：
- TAC 投稿（随时）
- ICMI 投稿（等截稿期）
- UMAP 投稿（等截稿期）

总时间：6-12个月
```

## 八、决策建议

### 如果你有以下情况：

**情况1：时间充足（6个月+），追求顶刊**

→ 投 ICMI + UMAP，等积累后再投 TAC
